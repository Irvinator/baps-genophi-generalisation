# GenoPHI Pipeline – Reproduction Guide

This guide explains exactly how to run the code and reproduce the results from the project.

There is only ONE correct order to run things.

---

# 1. WHAT THIS CODE DOES

This pipeline:

1. Takes a pre-built dataset of phage–host features  
2. Trains a machine learning model (CatBoost)  
3. Evaluates performance (ROC-AUC, PR-AUC, etc.)  
4. Produces plots and metrics used in the dissertation  

---

# 2. REQUIRED INPUT FILE (IMPORTANT)

Before running anything, make sure this file exists:
'data/reduced/final_merged_feature_table.csv'

This file is:
- already merged
- already cleaned
- ready for modelling

❗ You do NOT need to generate it  
❗ If it is missing, the pipeline will not run  

---

# 3. SETUP

Open a terminal and run:
  git clone <REPO_URL>
  cd <REPO_NAME>

  conda env create -f environment_hpc.yml
  conda activate genophi_hpc


---

# 4. RUN ORDER (FOLLOW EXACTLY)

Run each command one-by-one:

---

## Step 1 — Train / validation experiment
python scripts/scriptAA_train_val_pilot.py
### Expected output:

A new folder: pilot_train_val_outputs/

Inside this folder you should see:
- validation_metrics.csv  
- confusion_matrix.png  
- roc_curve.png  
- pr_curve.png  
- feature_importance.csv  
- trained model file (.cbm)

---

## Step 2 — Train final model
python scripts/scriptY_train_final_model.py

### Expected output: final_model_outputs/

Contains:
- final_metrics.csv  
- predictions.csv  
- confusion_matrix.png  
- feature_importance.csv  

---

## Step 3 — Cross-validation
python scripts/scriptZ_crossval_final_model.py
### Expected output: final_model_outputs_cv/

Contains:
- cross-validation metrics per fold  
- average performance summary  

---

## Step 4 — Feature importance analysis
python scripts/scriptAB_feature_importance.py

### Expected output:

- ranked feature importance table  
- optional plots  

---

## Step 5 — Feature coverage analysis
python scripts/scriptAD_feature_coverage.py

### Expected output:

- comparison of training vs validation feature presence  
- coverage statistics  

---

## Step 6 — Signal shift analysis
python scripts/scriptAE_signal_shift.py

### Expected output:

- analysis of how feature behaviour changes between datasets  

---

# 5. HOW TO KNOW IT WORKED

The pipeline has run successfully if:

- all folders listed above are created  
- CSV files contain numeric values  
- plots (ROC, PR, confusion matrix) are generated  

---

# 6. IMPORTANT NOTES

### Dataset

The dataset used here is a **reduced pilot dataset** due to computational limits.

It contains:
- 425 host genomes  
- 150 phage genomes  
- 474 interaction pairs  

---

### Negative samples

The original BAPS dataset contains **only positive interactions**.

Negative samples were generated by:
- pairing phages with hosts where no interaction is recorded  

This means:
- some negatives may actually be unknown positives  
- this introduces noise into the dataset  

---

### Performance interpretation

Observed:
- low ROC-AUC (~0.44 on external data)  
- model predicts mostly negatives  

This is expected due to:
- feature sparsity  
- poor overlap between training and external phages

# 7. COMMON ERRORS

---

## Error: file not found

Check: 
ls data/reduced/final_merged_feature_table.csv


If missing → place the file in that location

---

## Error: missing Python packages

Run: pip install catboost pandas numpy scikit-learn matplotlib


---

## Script crashes

Run scripts individually to debug:
python scripts/scriptAA_train_val_pilot.py


---

# 8. RUNNING ON HPC / MORE DATA

The current pipeline uses a **pre-built dataset**.

To run on larger datasets, you must:
- rebuild feature tables using MMseqs2
- regenerate merged dataset

This is computationally expensive and not required to reproduce results.
